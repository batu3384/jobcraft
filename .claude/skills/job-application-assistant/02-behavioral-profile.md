# Behavioral Profile

> Filled by `/setup` from self-assessment. Generic template until personalized.

## Overview
A **Builder–Analyst** pattern fits many software roles: shipping concrete products with clear structure, measurable outcomes, and ownership of a vertical slice.

## Core Behavioral Drives

| Drive | Level | Meaning |
|-------|-------|---------|
| Building / shipping | High | Energized by products and tools that reach users |
| Structure & clarity | High | Prefers defined goals, clean repos, reproducible setups |
| Security curiosity | Medium–High | Drawn to defensive tooling; calibrate claims to real depth |
| Social intensity / politics | Low–Medium | Prefers small collaborative teams |

## Strongest Behaviors
- **End-to-end ownership:** Problem from data/API through UI or CLI with runnable docs.
- **Learning by building:** Stacks learned through real projects.
- **Security-minded defaults:** Multi-tenant isolation, on-device processing where privacy matters.

## How You Work Best
- Clear backlog, async-friendly communication, defined location/remote preferences
- Mix of feature work and technical depth
- Mentorship for production hardening without micromanagement

## Growth Areas (frame positively in applications)
- **Experience depth:** Frame portfolio + willingness to grow under senior review.
- **Language level:** State honestly; note improvement path for international teams.

## Mapping to Job Posting Language

**Strong behavioral fit keywords:**
- full-stack, product-minded, ownership, shipping, API + UI
- security-conscious, DevSecOps-adjacent
- startup / small team, modern TypeScript/Python stack

**Potential friction (not always deal-breakers):**
- pure L1 support with no build time
- heavy travel / sales-only roles
- senior IC requirements with no junior track

## Management Style Preferences
- Works well with: transparent priorities, code review, pair on hard parts
- Doesn't work: unclear ownership, blame culture, endless meetings without decisions

## Using This in Applications
- **Cover letters:** Lead with shipped projects tied to the role.
- **CV:** Emphasize stack match without overselling seniority.
- **Interviews:** Use STAR examples in `07-interview-prep.md`.
- **Don't overstate:** Match claims to `01-candidate-profile.md` and `documents/`.
